# 溯造 MiniGame 宇宙观测站

“溯造 MiniGame 游戏开发大赛”的作品展示、探索与玩家之声投票网站。本届主题为“宇宙”，赛事标语为“溯求本源，造物不止”。

## 已实现体验

- 每次打开网站都会出现独立动态赛事海报，点击“开始观测”后以舱门动画进入星图
- 固定共享三维坐标的无边界星空，支持前后航行、侧移、旋转与主动进入全景观测
- 每台设备生成随机且不重复的自动浏览顺序，探索记录只保存在本机
- 远方作品以同等权重的异常恒星信号出现；靠近后依次显露颜色与类型、封面与玩法片段、名称、团队与作者，并保留已解析身份
- 探测器会同时解析视野内最近的三颗作品，屏幕边缘以克制的方向刻度与异常数量提示离屏信号
- 接近作品时自动减速并稳定在观测距离；偏离作品星域时无感校正航向，不出现空间边界
- 着陆后在同一空间展开作品名称、队伍、简介、创作手记、作品文件和制作成员卫星
- 次级搜索工具可直接按作品、队伍或成员名称定位
- 低性能与省流量设备保留完整三维导航语义，自动降低粒子、光晕和视频刷新成本
- 桌面与手机端均支持完整浏览和投票

## 投票与结果

- 游客可浏览全部内容，投票前使用姓名、队伍、邮箱和 6 位验证码登录
- 每人最多投 3 款作品，每票平等，截止前可随时增加、撤回或修改
- 投票以“提取可能性核心”仪式完成；核心进入私人个人星系，并在公开星空中留下仅自己可见的轨道标记
- 仪式结束后可继续探索、返回当前作品或留在个人星系，三个空间共享同一作品身份与选票状态
- 根据负责人和全部历史队友的已验证邮箱禁止自投，管理员可在后台人工复核并删除违规投票者
- 投票期内公开端不显示热度与票数，所有天体保持同等视觉权重
- 截止后只锁定选票并保持结果隐藏，不会自动公布
- 管理员完成同票裁定后手动触发“宇宙点亮”
- 固定两项同等级“玩家之声奖”，前五名组成星座，第五名同票时星座席位自动扩展
- 结果发布后，用户在下次刷新或重新打开时看到获奖恒星与星座

## 参赛者录入与内容管理

- 完成邮箱验证的参赛者可进入 `/submit`，使用“草稿、提交参展”流程创建作品
- 每款作品只有一个负责人，负责人可管理队友、提交与撤回；队友可编辑内容和自己的资料
- 作品禁止物理删除。撤回会隐藏作品、归还相关可能性核心并永久保留审计
- 上传本地游戏封面、最大 200MB 演示视频、最大 2GB 作品压缩包与最多 12 位成员头像，也可使用外部媒体地址
- 参展截止、投票截止与管理员手动宇宙点亮彼此独立
- 截止后提交、重新提交或替换作品文件会产生可审计的“补交”标记
- 配置活动名称、主题、标语、三个时间与固定星图种子
- 查看后台实时票数、完整素材元数据、成员权限、地址修改和审计记录，导出完整 CSV
- 删除违规投票者、处理获奖边界同票、手动发布最终结果

## 本地运行

```bash
npm install
npm start
```

公开星图：`http://localhost:3000/`

管理后台：`http://localhost:3000/admin`

参赛者工作台：`http://localhost:3000/submit`

本地默认后台密码为 `114514`。未配置 SMTP 且启用开发验证码时，验证码会显示在服务端终端并自动填入前端。

## 正式邮件配置

复制 `.env.example` 的配置到服务器环境文件，并填写 SMTP 信息：

```env
NODE_ENV=production
SMTP_HOST=smtp.example.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=sender@example.com
SMTP_PASS=邮箱服务商提供的授权码
MAIL_FROM="溯造 MiniGame <sender@example.com>"
OTP_SECRET=独立的长随机字符串
ALLOW_DEV_OTP=false
```

建议使用邮箱服务商的 SMTP 授权码，不要填写邮箱登录密码。

## 数据与上传目录

默认数据文件为 `data/minigame.json`，默认上传目录为 `data/uploads/`。生产环境应通过环境变量将二者放到代码目录外：

```env
MINIGAME_DATA_FILE=/var/lib/suyo-minigame/minigame.json
MINIGAME_UPLOAD_DIR=/var/lib/suyo-minigame/uploads
MAX_GAME_FILE_MB=2048
UPLOAD_PER_REQUEST_MBIT=150
# 设置为服务器实测入站带宽的 85%-90%，应用会在活跃上传之间动态均分；0 表示只使用单请求上限。
UPLOAD_TOTAL_MBIT=180
```

阿里云 Ubuntu 24.04 部署说明见 [部署文档](docs/deploy-aliyun-ubuntu-24.04.md)。公开投票涉及邮箱和身份信息，正式活动必须使用 HTTPS。
