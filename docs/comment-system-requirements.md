# 观测回声评论系统功能需求

## 1. 范围与术语

- 评论：直接针对作品发表的一级内容。
- 回复：属于某条一级评论的二级内容，可以指向一级评论或另一条回复。
- 共鸣：评论区内的点赞行为。
- 官方标签：由管理员维护并可聚合统计的标签。
- 自定义标签：用户为单条评论创建的短标签，不进入公共词表和聚合统计。
- 开发者：当前邮箱属于当前作品负责人或当前队友时，在该作品评论区实时显示的身份。

## 2. 权限矩阵

| 行为 | 游客 | 已验证用户 | 当前作品成员 | 评论管理员 |
| --- | --- | --- | --- | --- |
| 阅读公开评论 | 是 | 是 | 是 | 是 |
| 发表一级评论 | 否 | 是 | 自己作品下否，其他作品下是 | 是 |
| 回复 | 否 | 是 | 是 | 是 |
| 点赞与取消点赞 | 否 | 是 | 是 | 是 |
| 显示开发者身份 | 否 | 否 | 仅自己的作品 | 按真实关系查看 |
| 删除自己的内容 | 否 | 是 | 是 | 是 |
| 管理他人内容 | 否 | 否 | 否 | 是 |
| 管理官方标签 | 否 | 否 | 否 | 是 |
| 暂停评论区 | 否 | 否 | 否 | 是 |

## 3. 身份与头像

### FR-ID-01 发言认证

只有完成姓名、队伍与邮箱验证码验证的用户可以评论、回复和点赞。未登录用户只能阅读。

### FR-ID-02 公开身份

公开接口只返回当前姓名、公开头像和当前作品下的开发者状态，不返回邮箱和队伍。

### FR-ID-03 系统头像

系统依据规范化邮箱生成稳定且唯一的观测者头像。修改姓名不能改变默认头像。

### FR-ID-04 自定义头像

- 支持 JPG、PNG、WebP。
- 单个文件不超过 4MB。
- 服务端裁切或生成正方形公开版本。
- 删除自定义头像后恢复系统头像。
- 公开接口不能返回原始上传路径。

### FR-ID-05 统一资料

- 评论姓名复用全站登录身份姓名，不设置独立昵称。
- 在“观测者资料”修改姓名时，同步更新登录身份和历史评论前台显示。
- 如果用户属于某个作品，同步更新该作品中本人的成员姓名，队伍名称不变。
- 在作品提交页修改本人姓名时，同步更新全站身份与评论显示。
- 每次姓名与头像变更永久写入审计。

### FR-ID-06 开发者标识

- 仅当前作品的当前负责人和当前队友显示“开发者”。
- 在其他作品下不显示额外身份。
- 成员新增、移除、恢复或管理员纠正负责人后立即重新计算。
- 后台永久保存发表时的姓名、头像、邮箱、队伍和成员关系快照。

## 4. 评论与回复

### FR-COM-01 创建评论

- 一级评论最多 2,000 字。
- 支持换行和连续空格。
- 不解析 Markdown、HTML 或富文本，输出必须进行 HTML 转义。
- 允许只有文字或只有图片。
- 标签不能单独构成有效评论。
- 当前作品负责人和当前队友不能为自己的作品创建一级评论，但可以回复；在其他作品下按普通已验证用户处理。

### FR-COM-02 创建回复

- 回复最多 1,000 字。
- 回复可以指向一级评论或同楼层内另一条回复。
- 所有回复在数据上属于同一一级评论，界面不产生第三级缩进。
- 回复另一条回复时显示“回复 @当前姓名”。
- 用户可以回复自己。

### FR-COM-03 不可编辑

评论和回复发布后不允许编辑。用户需要补充内容时应回复自己。

### FR-COM-04 删除规则

- 内容没有任何直接或间接回复时，作者可从公开界面彻底移除。
- 内容已经产生回复时，作者删除只能转换为软删除占位。
- 所有删除都永久保留原文、图片元数据、作者、时间和操作记录供管理员查看。
- 删除或隐藏目标后，相关站内通知同步移除。

### FR-COM-05 排序和分页

- 一级评论按创建时间从新到旧。
- 回复完整视图按创建时间从旧到新。
- 详情预览默认返回最新 4 条一级评论，配置范围为 3-5 条。
- 完整空间每批默认 15 条，配置范围为 10-20 条。
- 使用游标分页，不能使用容易在新增评论时重复或遗漏的页码偏移。
- 自动加载失败时必须允许手动重试。

### FR-COM-06 默认回复预览

每条一级评论默认只嵌套最早一条仍公开可见的回复，并返回其余可见回复数量。点击后加载该楼层完整回复。

### FR-COM-07 频率限制

- 同一邮箱两次创建评论或回复之间至少间隔 10 秒。
- 每个邮箱按北京时间自然日最多创建 100 条评论和回复。
- 相同用户对同一作品或同一楼层在短时间提交相同规范化正文、图片集合与标签集合时，只创建一次。
- 重复请求应返回第一次创建的内容标识，避免前端错误显示为失败。

## 5. 图片

### FR-IMG-01 文件规则

- 一级评论和回复最多各 5 张图片。
- 支持 JPG、PNG、WebP。
- 不支持 GIF 和 SVG。
- 单张原图不超过 10MB。

### FR-IMG-02 存储与派生文件

- 保留原图，并生成适合列表加载的缩略图。
- 永久记录原文件名、MIME、大小、哈希、宽高、上传者和上传时间。
- 原图与缩略图使用不可猜测存储名。
- 公开接口返回受控资源地址，不返回本地文件路径。

### FR-IMG-03 上传状态

- 前端显示每张图片的独立进度。
- 单张失败不取消其他图片。
- 失败图片允许单独重试或移除。
- 所有图片成功上传或被移除前不能发布评论。
- 系统不保存跨页面未发布草稿。

### FR-IMG-04 原图查看

点击缩略图后打开原图查看器，支持同一评论内前后切换、缩放、关闭和键盘操作。

### FR-IMG-05 管理员移除图片

管理员可以单独移除违规图片并填写原因。公开页面保留“图片已由管理员移除”占位，文字与其他图片继续显示。原文件和处理记录按审计策略保留在管理员范围内。

## 6. 标签

### FR-TAG-01 使用范围

- 只有一级评论可以使用标签。
- 每条评论可选择 0-5 个标签。
- 标签可由官方标签和自定义标签混合组成。

### FR-TAG-02 自定义标签

- 每个自定义标签最多 8 个 Unicode 字符。
- 规范化首尾空格并拒绝空标签。
- 仅随当前评论展示，不加入公共标签库，不进入聚合统计。
- 管理员可以单独移除违规自定义标签并填写原因。

### FR-TAG-03 官方标签管理

- 管理员可以新增、排序、启用和停用官方标签。
- 停用标签不能再被新评论选择。
- 已经使用的停用标签继续显示在历史评论上。
- 删除或隐藏评论后，其官方标签不计入统计；恢复后重新计入。

### FR-TAG-04 标签统计

后台按作品展示官方标签的使用次数和时间趋势。自定义标签不进入该统计。

## 7. 点赞

### FR-LIKE-01 共鸣行为

- 一级评论和回复均可点赞。
- 同一邮箱对同一内容最多存在一个有效点赞。
- 用户可以随时取消点赞。
- 用户不能点赞自己的评论或回复。
- 点赞数量公开展示，但不参与评论排序或作品排序。

### FR-LIKE-02 并发一致性

点赞和取消点赞必须使用数据库唯一约束或等价原子机制，避免快速点击产生重复记录或负数计数。

## 8. 通知

### FR-NOT-01 回复通知

回复创建成功后通知被回复者。回复自己时不创建通知。

### FR-NOT-02 点赞通知

- 点赞通知按目标内容聚合。
- 示例文案：“有 5 人共鸣了你的评论”。
- 用户取消点赞后聚合数量应反映当前有效点赞数。
- 聚合数量变为 0 时移除该通知。

### FR-NOT-03 作品新评论通知

- 当前作品成员可以分别开启或关闭“本作品新评论”。
- 默认关闭。
- 开启后只通知启用期间创建的新一级评论。
- 成员被移除后立即停止接收。
- 成员重新加入后不补发历史通知。

### FR-NOT-04 通知偏好

用户可以独立关闭回复通知、点赞通知和作品新评论通知。

### FR-NOT-05 通知中心

- 展示未读数量。
- 支持单条已读与全部已读。
- 点击后打开对应作品、对应一级评论和回复，并短暂高亮目标。
- 稳定定位只供站内通知使用，不提供复制链接入口。
- 目标被作者删除或管理员隐藏后，相关用户通知移除。

## 9. 作品与活动生命周期

### FR-LIFE-01 开放时机

作品提交并公开后立即开放评论。评论长期开放，不受投票开始、投票结束和宇宙点亮影响。

### FR-LIFE-02 作品撤回

- 作品撤回后评论区进入只读状态。
- 公开作品入口消失后，已有合法访问上下文仍只允许读取，不允许新增评论和回复。
- 作品重新提交后原评论与回复恢复可交流状态。

### FR-LIFE-03 管理员暂停

- 管理员可以将全站评论设为只读。
- 管理员可以将单个作品评论区设为只读。
- 暂停和恢复必须填写原因并写入审计。

### FR-LIFE-04 活动归档

活动归档时，评论、回复、点赞、标签、图片和管理记录随活动进入历史数据。旧活动评论只供管理员查询，公开前台不再访问。

### FR-LIFE-05 作品导出

评论、回复、评论图片、点赞和评论通知不加入单作品或全部作品 ZIP 导出。

## 10. 管理与治理

### FR-MOD-01 发布策略

评论和回复发布后立即公开，不设置审核队列、敏感词待审或用户举报入口。

### FR-MOD-02 开发者权限

作品成员在自己的作品下只能以开发者身份回复，不能创建一级评价，也不能隐藏、恢复、删除、折叠、置顶、标记或管理任何用户内容。在其他作品下按普通已验证用户处理。

### FR-MOD-03 内容管理

管理员可以：

- 隐藏单条评论或回复。
- 恢复被隐藏内容。
- 单独移除违规图片。
- 单独移除违规自定义标签。
- 查看用户全部评论、回复、点赞与管理状态。

隐藏一级评论或回复时保留其讨论结构。普通用户看到“这段信号暂不可见”，管理员看到原内容和处理信息。

### FR-MOD-04 评论禁言

- 支持 1 小时、1 天、7 天和永久禁言。
- 禁言只阻止新评论和新回复。
- 禁言用户仍可阅读、点赞、取消点赞、管理通知和删除自己的历史内容。
- 管理员可以提前解除禁言。
- 评论禁言不影响浏览、投票、作品创建和作品编辑。

### FR-MOD-05 管理审计

隐藏、恢复、删图、移除自定义标签、禁言、解禁、全站暂停、作品暂停和恢复都必须填写原因，并永久记录操作者、目标、前后状态和北京时间。

## 13. 安全、性能与隐私

- 正文只按纯文本存储和转义渲染，禁止执行 HTML。
- 文件上传验证扩展名、MIME、文件头、尺寸和图片解码结果。
- 删除 EXIF 和其他不必要元数据后生成公开版本。
- 对创建、点赞、头像与图片上传分别限速。
- 使用游标和索引支持 `game_id + status + created_at`、`comment_id + status + created_at` 查询。
- 列表只加载缩略图，原图按用户操作加载。
- 公开缓存不能包含邮箱、队伍、身份快照和管理员状态。
- 评论内容与投票结果相互独立，评论点赞不能影响星图权重和奖项。
- 所有时间判断和后台展示使用北京时间并精确到秒。

## 14. 验收标准

1. 游客可以浏览评论，但任何写操作都会引导邮箱验证。
2. 已验证用户可以发布纯文本、仅图片或图文评论，并遵守数量与大小限制。
3. 一级评论最多 5 个标签，自定义标签不进入公共词表和官方统计。
4. 默认详情只显示 4 条一级评论，每条最多预览一条回复。
5. 完整讨论空间可以稳定自动加载，新增评论不会导致重复或遗漏。
6. 评论和回复都能点赞、取消点赞，不能自赞，点赞不改变排序。
7. 当前作品成员只在本作品显示“开发者”，成员变更后立即更新。
8. 评论发布后不能编辑，可以回复自己补充。
9. 删除有回复内容时保留占位，管理员始终可以查看原始内容与审计。
10. 回复和点赞通知按规则生成、聚合、关闭和定位，目标隐藏后通知消失。
11. 作品撤回后评论只读，重新提交后恢复；投票和宇宙点亮不影响评论。
12. 管理员能隐藏、恢复、删图、移除自定义标签及临时或永久禁言，所有操作要求原因。
13. 评论发布、点赞和通知定位保持静音，并支持减少动态效果。
14. 活动归档后旧评论只在管理员后台可查，不出现在新活动公开页面。
15. 单作品和全部作品 ZIP 导出不包含任何评论数据。
