# 溯造 MiniGame：可能性核心投票仪式产品需求

状态：需求已确认
版本：1.1
日期：2026-07-14
关联设计文档：`docs/possibility-core-voting-ritual-design.md`

## 1. 产品目标

将现有即时投票操作升级为具有空间仪式感、明确二次确认且不牺牲可修改性的投票流程，并确保它从作品星空进入、在个人星系完成、再沿原观测航迹返回，构成同一宇宙中的连续体验。

流程必须保证：

1. 用户点击投票后不会立即写入选票。
2. 用户持续长按三秒完成第一次确认。
3. 用户主动点击提取出的核心完成第二次确认。
4. 核心只有在服务端确认选票后才进入个人轨道。
5. 投票截止前仍可撤回和替换。
6. 视觉状态必须与服务端真实选票状态一致。
7. 公共星空、个人星图和个人星系必须视觉连续、数据隔离。

## 2. 范围

本需求包括：

- 新增选票仪式。
- 满三票后的替换仪式。
- 个人星系与用户行星。
- 撤回和归还流程。
- 核心光谱、粒子、引力波、声音与触觉反馈。
- 网络重试、版本冲突与失败恢复。
- 桌面端、移动端、键盘和减少动态效果模式。
- 公共作品星空与个人星系之间的连续转场、返回航迹和私人投票标记。

本需求不包括：

- 修改每人最多三票和三票等权规则。
- 公开个人星系或用户选择。
- 根据最终获奖结果改变个人星系。
- 展示实时票数、作品热度或他人选票。
- 建立新的长期账号或社交关系系统。

## 3. 术语

| 术语 | 定义 |
| --- | --- |
| 可能性核心 | 从被投作品行星中提取的抽象光谱能量核 |
| 用户行星 | 由当前用户姓名与队伍稳定生成的私人天体 |
| 个人星系 | 用户查看和管理自己选票的私人空间 |
| 轨道槽位 | 最多三个完全等权的有效选票位置 |
| 悬浮核心 | 已完成提取但尚未获得服务端确认的核心 |
| 入轨 | 服务端确认新增选票后，核心进入轨道的状态 |
| 交接 | 三票已满时，新旧核心在轨道边缘等待原子替换的状态 |
| 归还 | 服务端确认撤票后，核心离开个人星系的状态 |
| 公共星空 | 所有用户共享的固定作品三维坐标与公共结果空间 |
| 个人星图 | 当前设备已解析状态与当前登录者私人投票标记形成的本地视图 |

## 4. 业务规则

| 规则 | 要求 |
| --- | --- |
| 票数 | 每位已验证用户最多三票 |
| 票权 | 三票完全平等，不排序、不加权 |
| 生效时点 | 服务端确认后核心入轨，选票生效 |
| 提取时点 | 提取完成不等于选票生效 |
| 修改期限 | 仅投票状态为 OPEN 时允许新增、替换和归还 |
| 自投 | 姓名匹配作品制作人员时禁止启动仪式 |
| 公开性 | 个人星系与用户行星仅本人可见 |
| 结果影响 | 最终结果不改变个人星系的核心样式和亮度 |
| 超时 | 提交十五秒无确认时停止重试、归还核心并退出 |

## 5. 客户端状态机

```text
IDLE
  -> AUTH_REQUIRED
  -> READY
  -> EXTRACTING
  -> READY                 松开，进度回退
  -> CORE_FLOATING         三秒提取完成
  -> SUBMITTING            有空槽，点击核心
  -> REPLACEMENT_SELECT    三槽已满，点击核心
  -> HANDOFF               选择旧核心
  -> SUCCESS_ORBIT         服务端确认
  -> PERSONAL_SYSTEM       观察约四秒
  -> RETURNING_PUBLIC      继续探索或返回作品
  -> IDLE                  恢复公共作品上下文

SUBMITTING 或 HANDOFF
  -> RETRYING              暂无响应，十五秒内重试
  -> SUCCESS_ORBIT         服务端确认成功
  -> RETURNING             十五秒仍无确认
  -> IDLE                  核心返航并退出

CORE_FLOATING 或 REPLACEMENT_SELECT
  -> RETURNING             用户取消、返回或关闭
  -> IDLE
```

## 6. 功能需求

### 6.1 仪式入口

#### VR-ENTRY-01 延迟登录

未登录用户点击“投出一票”时，系统先完成邮箱验证码登录。登录成功后必须恢复原作品上下文并进入投票仪式。

#### VR-ENTRY-02 启动前校验

启动仪式前必须校验：

- 当前投票状态为 OPEN。
- 用户会话有效。
- 作品存在且公开。
- 当前选票尚未包含该作品。
- 用户姓名未命中作品制作人员姓名。

校验失败时不得进入三秒提取。

#### VR-ENTRY-03 无即时写入

点击“投出一票”和完成三秒提取都不得写入服务端选票。只有用户完成第二次确认后才发起选票变更。

### 6.2 三秒提取

#### VR-EXTRACT-01 持续时间

提取必须要求连续按住三秒。进度以单调递增的可见数值或图形反馈表示。

允许输入：

- 鼠标主键持续按住。
- 单指触控持续按住。
- 键盘空格键持续按住。
- 键盘 Enter 键持续按住。

#### VR-EXTRACT-02 中断回退

三秒完成前释放对应输入时：

- 停止提取。
- 进度回退到零。
- 引力波与粒子恢复初始状态。
- 停止本次渐进声音和触觉计划。
- 不创建悬浮核心。
- 不发送选票写入请求。

#### VR-EXTRACT-03 稳定操作层

以下元素在提取过程中不得扭曲、模糊或位移：

- 长按区域。
- 提取进度。
- 作品名称和队伍。
- 当前操作说明。
- 错误与取消信息。

背景星场、作品媒体、作品行星、坐标网格和非关键边缘界面可以逐渐扭曲。

#### VR-EXTRACT-04 粒子反馈

粒子随进度从作品区域向提取点汇聚。粒子密度与速度可以分性能等级，但不得完全移除提取方向和进度关系。

#### VR-EXTRACT-05 完成反馈

三秒完成时：

- 核心脱离作品行星。
- 播放一次明确的完成声音。
- 支持的手机设备提供一次明确触觉反馈。
- 镜头开始从作品行星拉远并显露个人星系。

### 6.3 可能性核心

#### VR-CORE-01 视觉来源

核心颜色和纹理由作品封面、演示视频、作品编号和天体种子共同生成。必须保留来源作品的光谱特征。

#### VR-CORE-02 统一调色

系统必须限制核心最大饱和度和明度范围，使三颗来源不同的核心在同一私人星系中保持和谐，并维持三票视觉等权。

#### VR-CORE-03 抽象形态

核心必须由光谱压缩、引力透镜和粒子包络构成。不得使用宝石切面、抽卡稀有度、魔法球、奖章或火焰造型。

#### VR-CORE-04 悬浮状态

核心完成提取后悬浮在个人星系外侧。悬浮状态不表示有效选票。

用户在悬浮状态执行关闭、返回或取消时，核心立即返航，且不得修改选票。

### 6.4 镜头与个人星系显露

#### VR-CAMERA-01 连续拉远

提取完成后必须沿当前背景星场使用连续空间转场，从作品行星拉远至个人星系。不得使用无关联的页面硬切或普通弹窗替代。

#### VR-CAMERA-01A 公共上下文快照

进入个人星系前必须保存：

- 公共摄像机位置与朝向。
- 自由航行或全景观测模式。
- 当前观测目标。
- 来源作品 ID。
- 进入前处于作品详情还是观测壳层。

私人星系坐标不得覆盖公共摄像机快照。

#### VR-CAMERA-02 身份可见

个人星系显露时必须显示当前用户姓名和队伍，使用户明确识别这是自己的私人空间。

#### VR-CAMERA-03 减少动态效果

减少动态效果模式下可以用缩放、透明度和短距离位移替代大幅镜头变化，但作品空间到个人空间的状态关系必须保留。

### 6.5 第二次确认与空槽入轨

#### VR-PLACE-01 空间确认

系统不得再显示传统“确定投票吗”确认框。第二次确认由用户主动点击悬浮核心完成。

#### VR-PLACE-02 自动选择空槽

存在空槽时，用户点击核心后，系统自动选择最近的空槽位。不得要求用户为等权选票选择第一、第二或第三位置。

#### VR-PLACE-02A 接入点一致性

引导航迹、接入点预览、飞行动画终点与核心开始公转的位置必须由同一套轨道几何参数计算。星系缩放、窗口尺寸变化或响应式布局不得造成预览位置与实际落点偏离。

#### VR-PLACE-03 提交前位置

发起服务端请求后，核心必须停留在轨道外等待。服务端未确认前不得播放完整入轨和成功反馈。

#### VR-PLACE-04 生效反馈

服务端确认后：

- 核心进入空槽并开始公转。
- 当前作品加入有效选票集合。
- 页面显示“选票已生效”和当前已使用槽位数量。
- 用户行星吸收核心的光谱特征并增加活跃度。
- 支持的手机设备提供一次明确触觉反馈。

### 6.6 满槽替换

#### VR-REPLACE-01 新核心悬停

三槽已满时，新核心完成提取后必须停留在个人星系外，不得要求用户先撤回旧票。

#### VR-REPLACE-02 选择旧核心

用户必须亲自从三颗旧核心中选择一颗进行替换。选择界面必须显示旧核心对应的游戏名称和队伍。

#### VR-REPLACE-03 轨道边缘交接

用户选择旧核心后，新旧核心移动到轨道边缘形成短暂交接。旧核心在服务端确认前仍属于有效选票。

#### VR-REPLACE-04 原子更新

服务端必须以一次原子操作完成：

- 移除被选旧作品。
- 加入新作品。
- 保持选票总数不超过三。
- 校验自投、作品状态、会话状态和投票截止时间。

任何条件失败时不得只执行其中一部分。

#### VR-REPLACE-05 成功与失败

- 成功时旧核心返航，新核心进入原槽位。
- 失败时旧核心回到原槽位，新核心恢复悬浮等待状态或按超时规则返航。
- 版本冲突时先读取服务端最新选票，再重新构建个人星系，不得静默覆盖另一设备的修改。

### 6.7 五秒上下文提示

#### VR-HINT-01 触发时间

用户在当前仪式状态连续五秒没有有效输入时，显示一条与当前状态匹配的操作提示。

#### VR-HINT-02 状态匹配

提示至少覆盖：

- 尚未长按。
- 长按中断。
- 核心等待点击。
- 满槽后等待选择旧核心。
- 网络等待。
- 个人星系中等待选择核心。

#### VR-HINT-03 非阻塞

提示不得自动执行操作、抢夺键盘焦点或遮挡核心。用户恢复操作后提示自动隐藏。

### 6.8 成功余韵

#### VR-SUCCESS-01 日常轨道速度

新核心入轨后的初始公转速度可以略高于日常速度，但不得出现快速补完半圈或整圈。建议初始不超过日常速度的 1.25 倍，并在第一圈内连续、平滑地衰减到日常速度；速度切换不得造成位置跳变。

#### VR-SUCCESS-02 四秒停留

成功后镜头在个人星系停留约四秒，让用户观察核心和用户行星的变化。四秒后显示可操作的“继续探索”“返回作品”和“留在个人星系”。

#### VR-SUCCESS-02A 返回目标

- “继续探索”返回来源作品的公共观测壳层，并恢复合理朝向。
- “返回作品”回到来源作品详情。
- “留在个人星系”不得自动启动返回计时。
- 返回转场可以连续或在减少动态模式下简化，但不得丢失来源作品。

#### VR-SUCCESS-03 不显示热度

成功状态只显示个人选票容量，不得显示作品公开票数、排名、百分比或热度暗示。

### 6.9 用户行星

#### VR-USER-01 稳定生成

用户行星以标准化后的姓名与队伍生成稳定种子。同一身份在不同设备登录时基础形态一致。

#### VR-USER-02 身份标识

个人星系显示姓名和队伍。邮箱不得进入可见形态、标签、分享内容或前端生成种子。

#### VR-USER-03 核心注入

每颗入轨核心向用户行星贡献来源作品的光谱特征。贡献必须经过统一调色，不能直接把封面贴到行星表面。

#### VR-USER-04 累积生机

用户行星随零到三颗核心逐步增加亮度、表面活动与周边粒子生机。三颗核心的贡献必须等权。

#### VR-USER-05 私人数据

用户行星形态、姓名、队伍和当前核心组合仅当前登录用户可见。公开接口和公共星图不得返回或展示这些数据。

### 6.10 个人星系

#### VR-SYSTEM-01 独立入口

已登录用户可以随时从全站打开个人星系。未登录用户打开入口时进入登录流程。

#### VR-SYSTEM-02 核心浏览

核心默认不常驻显示游戏名称。用户靠近、悬停、键盘聚焦或点击核心时，显示游戏名称、队伍和可用操作。

#### VR-SYSTEM-03 作品跳转

用户可以从核心返回对应作品行星或作品详情。跳转到作品行星时必须使用公共星空导航并到达该作品固定坐标；退出个人星系后恢复保存的公共摄像机状态或明确的新目标观测壳层。

#### VR-SYSTEM-04 归还入口

OPEN 状态下，用户可以从个人星系选择核心并发起归还。投票截止后入口禁用并解释原因。

#### VR-SYSTEM-05 结果独立

结果发布不得改变个人星系中任何核心的亮度、大小、轨道或光谱。个人星系不标记获奖或落选。

### 6.11 归还与撤回

#### VR-RETURN-01 个人星系归还

用户选择旧核心后，使用一次直接的“归还”确认。归还不要求三秒长按。

#### VR-RETURN-02 作品详情撤回

从作品详情点击“撤回这一票”时，可以使用更直接的确认流程，不强制先打开个人星系。

#### VR-RETURN-03 服务端优先

只有服务端确认撤票后，才播放完整返航并移除轨道核心。请求失败时核心保持原轨道状态。

#### VR-RETURN-04 行星回退

归还成功后，用户行星移除该核心贡献的光谱和活跃度，但保留其他核心的贡献。

### 6.11.1 公共星空私人标记

#### VR-MARK-01 生效时点

只有服务端确认新增或替换成功后，当前登录用户的公共星空才增加对应作品的私人投票标记。核心仍在悬浮、提交或交接状态时不得提前显示。

#### VR-MARK-02 私密与等权

- 标记仅当前登录用户可见。
- 标记不得进入公共接口、分享卡片、结果分享图或其他用户界面。
- 标记不得改变作品尺寸、亮度、探测半径、布局、搜索排序或自动巡航顺序。

#### VR-MARK-03 撤回与替换

- 撤回只有在服务端确认后才移除标记。
- 替换只有在原子更新确认后才同时移除旧标记并增加新标记。
- 请求失败、超时或版本冲突时，标记必须按服务端最新选票重建。

#### VR-MARK-04 结果独立

公共结果发布可以改变公共获奖恒星和星座，但不得改变私人投票标记与个人星系核心之间的对应关系。

### 6.12 网络重试与一致性

#### VR-NET-01 操作标识

每次新增或替换生成唯一 `operationId`。十五秒内的全部自动重试必须使用同一个标识，服务端必须实现幂等处理。

#### VR-NET-02 重试窗口

服务端没有回应时，客户端在十五秒内进行多次重试。重试期间核心停留在轨道外，旧核心保持有效。

#### VR-NET-03 超时返航

十五秒仍无确认时：

- 停止重试。
- 悬浮核心返航。
- 恢复被选旧核心的正常轨道状态。
- 退出仪式。
- 显示“信号未能锁定，核心已经返回”。

#### VR-NET-04 后续对账

网络恢复或下次打开个人星系时，客户端必须重新读取服务端选票。若超时操作实际已在服务端提交，客户端以服务端为准恢复核心并说明选票已同步。

#### VR-NET-05 版本冲突

请求必须携带当前选票版本。服务端发生版本冲突时返回最新选票和版本，客户端重建个人星系并要求用户重新确认目标变更。

### 6.13 声音

#### VR-AUDIO-01 用户手势后播放

仪式声音只能在明确用户手势后播放，遵守浏览器自动播放限制。

#### VR-AUDIO-02 状态对应

至少提供：

- 渐进提取底音。
- 核心脱离确认音。
- 入轨确认音。
- 低存在感的网络等待信号。

#### VR-AUDIO-03 可关闭

提供可访问的静音控制。静音后所有状态仍通过视觉和文字完整表达。

### 6.14 触觉反馈

#### VR-HAPTIC-01 渐进提取

支持触觉反馈的手机设备在三秒提取过程中提供由轻到深的反馈。

#### VR-HAPTIC-02 关键节点

核心脱离行星和成功入轨时各提供一次更明确反馈。

#### VR-HAPTIC-03 渐进增强限制

触觉反馈应短促、可中断，不得形成持续强震。用户松开、取消、切换页面或仪式结束时必须停止未完成的触觉计划。

#### VR-HAPTIC-04 能力降级

设备或浏览器不支持触觉时静默降级，不能报错，也不能影响投票。

### 6.15 减少动态效果与性能分级

#### VR-PERF-01 表现等级

完整、标准和简化二维模式必须拥有相同的投票状态机和业务结果。

#### VR-PERF-02 减少动态效果

减少动态效果模式必须：

- 保留三秒提取。
- 保留清晰进度。
- 保留数量降低的粒子效果。
- 保留核心悬浮、点击、入轨、交接和返航状态。
- 移除强烈引力扭曲。
- 缩短或简化大幅镜头运动。

#### VR-PERF-03 生命周期

关闭仪式、退出个人星系、页面隐藏或组件销毁时，必须停止对应动画帧、计时器、音频节点和触觉任务。

### 6.16 无障碍

#### VR-A11Y-01 键盘完成

用户必须能够仅用键盘完成提取、置入、替换、归还和退出。

#### VR-A11Y-02 状态播报

辅助技术至少能够读取：

- 当前目标作品。
- 提取进度。
- 提取已完成但选票尚未生效。
- 当前轨道容量。
- 正在提交、成功、失败和超时状态。
- 替换目标。

#### VR-A11Y-03 不依赖单一感官

任何状态不得只用颜色、声音、粒子或触觉表达。

#### VR-A11Y-04 焦点管理

仪式开始时焦点进入长按区域。核心出现后焦点移动或可预测地前进到核心。仪式结束后焦点返回触发入口或合理的继续探索入口。

## 7. 数据与接口要求

### 7.1 用户行星种子

客户端或服务端使用以下标准化输入生成稳定种子：

```text
normalize(name) + separator + normalize(team) + eventSeed
```

邮箱不得参与视觉种子。不同赛事必须通过 `eventSeed` 生成不同基础宇宙。

### 7.2 核心视觉数据

每款作品需向前端提供或可推导：

- 稳定作品 ID。
- 行星种子。
- 封面地址。
- 演示视频地址。
- 预计算或运行时提取的主光谱色。
- 低饱和辅助光谱色。

### 7.3 选票变更语义

新增和替换请求至少包含：

- `operationId`。
- `expectedBallotVersion`。
- `addGameId`。
- 替换时的 `removeGameId`。

成功响应至少包含：

- 最新有效作品 ID 集合。
- 最新选票版本。
- 服务端确认时间。
- 已处理的 `operationId`。

### 7.4 公开数据边界

公开赛事接口不得包含：

- 用户姓名、队伍或邮箱。
- 个人星系种子。
- 用户核心组合。
- 未发布票数和排名。
- 可反推出个人选票的数据。

私人投票标记不得由公开作品数据直接返回。登录客户端只能根据受保护的当前选票接口在本地构建标记。个人星系坐标和公共摄像机快照必须存储在不同状态域中。

## 8. 建议埋点

埋点仅用于体验质量，不记录公开选票内容。

| 事件 | 目的 |
| --- | --- |
| `vote_ritual_started` | 计算仪式进入率 |
| `vote_extraction_released` | 判断三秒长按是否过难 |
| `vote_core_extracted` | 计算第一次确认完成率 |
| `vote_core_cancelled` | 判断悬浮阶段反悔率 |
| `vote_core_submitted` | 计算第二次确认完成率 |
| `vote_replacement_started` | 观察满槽替换使用情况 |
| `vote_network_timeout` | 监控十五秒超时问题 |
| `vote_core_orbited` | 确认成功体验完成率 |
| `personal_system_opened` | 评估私人空间价值 |

埋点不得包含邮箱原文、验证码、完整姓名或其他无需收集的个人信息。

## 9. 验收标准

### AC-VR-01 新增选票

已登录用户点击投票，持续按住三秒，点击悬浮核心。服务端确认后核心进入最近空槽，选票集合增加该作品，用户行星出现对应光谱变化。

### AC-VR-02 长按中断

用户在 2.9 秒松开，进度回退，粒子和扭曲恢复，不生成悬浮核心，不发送写请求。

### AC-VR-03 提取后取消

用户完成提取但未点击核心，随后关闭仪式。核心立即返航，选票不变。

### AC-VR-04 五秒引导

核心悬浮五秒无操作后，页面提示点击核心入轨。用户点击后提示消失，不需要关闭弹窗。

### AC-VR-05 满槽替换

用户已有三票，提取第四款核心并选择一颗旧核心。新旧核心在轨道边缘交接，服务端原子替换成功后旧核心返航、新核心入轨，选票总数仍为三。

### AC-VR-06 替换失败

替换请求失败时，旧核心仍在原轨道，新核心不得假装入轨，服务端选票保持原集合。

### AC-VR-07 网络超时

新增或替换十五秒没有任何确认时，停止自动重试，核心返航并退出仪式。网络恢复后客户端重新同步服务端选票。

### AC-VR-08 跨设备冲突

设备 A 与设备 B 同时修改同一选票时，其中一个请求发生版本冲突。冲突设备读取最新选票并重建个人星系，不静默覆盖成功修改。

### AC-VR-09 归还

用户在个人星系点击旧核心并确认归还。服务端确认后核心返航、槽位清空、用户行星移除对应光谱贡献。

### AC-VR-10 归还失败

撤票请求失败时，核心继续公转，选票不变，页面允许重试。

### AC-VR-11 个人星系隐私

未登录访客、其他用户和公开 API 均不能读取当前用户的个人星系、姓名、队伍或核心组合。

### AC-VR-12 结果发布

全站宇宙点亮后，个人星系保持点亮前的核心亮度、光谱和轨道，不标记获奖或落选。

### AC-VR-13 减少动态效果

开启减少动态效果后，用户仍能通过三秒进度、少量粒子、文字和核心位置变化完成新增、替换与归还，不出现强烈页面扭曲。

### AC-VR-14 键盘流程

用户只使用键盘即可持续提取三秒、点击核心、选择替换目标、确认归还和继续探索。焦点顺序可预测，状态有文字播报。

### AC-VR-15 性能与清理

连续打开和关闭仪式十次后，不产生重复轨道循环、叠加音频、残留触觉任务或不断增长的计时器数量。

### AC-VR-16 星空连续往返

用户从作品行星详情完成提取并进入个人星系。成功入轨后选择“继续探索”，镜头返回来源作品公共观测壳层，位置、朝向和目标合理；选择“返回作品”则回到来源详情，不出现无关页面硬切。

### AC-VR-17 私人标记一致性

新增成功前公共星空不显示私人标记；服务端确认后只在当前用户界面显示。撤回失败时标记和核心都保留，原子替换成功时旧标记与新标记同步交换。另一未登录设备和公开接口均无法读取标记。

### AC-VR-18 公私结果分离

公共玩家之声结果发布后，获奖作品在公共星空中点亮；个人星系核心和私人投票标记不因获奖或落选改变亮度、尺寸、光谱或轨道。

## 10. 上线前检查

- 验证三秒长按在鼠标、Android 触控、iOS 触控和键盘下行为一致。
- 验证页面滚动与浏览器长按菜单不会抢占提取手势。
- 验证三种性能等级的状态机完全一致。
- 验证静音和不支持触觉时仍有完整反馈。
- 验证十五秒超时、断网恢复和请求响应丢失场景。
- 验证三票满槽替换为一次原子更新。
- 验证投票截止瞬间正在进行的仪式不会越权提交。
- 验证个人星系不进入公开接口、分享卡片和搜索索引。
- 验证私人投票标记只从受保护的当前选票构建，不进入公开作品数据。
- 验证从任意作品详情进入仪式后，三个返回去向都能恢复正确公共上下文。
- 验证三颗高饱和来源核心经过统一调色后仍然和谐。
- 验证结果发布不会改变个人星系。
