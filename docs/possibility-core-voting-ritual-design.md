# 溯造 MiniGame：可能性核心投票仪式设计文档

状态：交互方向已确认
版本：1.1
日期：2026-07-14
关联文档：`docs/cosmic-observatory-design.md`、`docs/possibility-core-voting-ritual-requirements.md`

## 1. 设计结论

投票不再是点击按钮后播放一段装饰动画，也不是脱离星空的独立页面，而是一套由用户从公共作品宇宙进入私人星系、再返回探索航迹的连续空间行为：

1. 从作品行星中持续提取一颗“可能性核心”。
2. 看着镜头从作品行星拉远，显露自己的私人星系。
3. 主动点击悬浮核心，让它进入三个等权轨道槽位之一。
4. 核心成功入轨后，用户行星获得这款作品的光谱与生机。
5. 用户沿保存的公共观测航迹返回来源作品，或从个人星系前往任意已投作品。

这套行为本身承担二次确认，不再叠加普通的“确定投票吗”弹窗。

第一层确认是持续三秒长按完成提取。第二层确认是主动点击悬浮核心完成置入。轨道已满时，第二层确认变为亲自选择一颗旧核心完成交接。

## 2. 设计判断

这是一次既有投票流程的定向重设计。它面向已经完成作品探索、准备表达支持的玩家。设计语言延续真实天文观测与数字展览，不使用宝石、魔法球、抽卡或儿童宇宙语言。

设计参数：

- `DESIGN_VARIANCE: 8`：仪式采用空间转场与非标准确认方式，但不改变全站信息结构。
- `MOTION_INTENSITY: 8`：镜头、粒子、引力波、轨道与触觉共同表达选票状态。
- `VISUAL_DENSITY: 4`：仪式中只保留必要的作品身份、提取进度、轨道容量和操作提示。

动效的理由是表达状态：提取表示用户正在确认，悬浮表示尚未提交，入轨表示选票生效，返航表示撤回，交接表示原子替换。

## 3. 核心体验命题

### 3.1 一张票是什么

一张票不是排名分数，而是探索者从作品行星中识别出的一种玩法可能性。

可能性核心由作品的光谱、引力透镜和粒子凝聚而成。它没有坚硬宝石表面，也不表现为魔法能量球。它更接近一个可被观测、但无法被完全解释的高密度信号。

### 3.2 投票为什么有重量

重量来自三个事实：

1. 用户必须持续三秒稳定提取，不能误触完成。
2. 提取完成仍不自动提交，用户需要再次主动点击核心。
3. 用户只有三个等权轨道槽位，轨道已满时必须亲自完成新旧选择的交接。

### 3.3 可撤回不等于轻率

截止前选票始终可以撤回和替换。仪式表达的是“我此刻愿意把这份可能性带入自己的星系”，不暗示不可逆承诺。

撤回称为“归还”。核心被归还作品行星，用户行星同时失去由该核心注入的部分光谱与活跃度。

## 4. 概念映射

| 产品状态 | 空间隐喻 | 视觉动作 |
| --- | --- | --- |
| 准备投票 | 锁定可能性信号 | 作品行星进入稳定观测状态 |
| 长按确认 | 提取可能性核心 | 粒子向提取点汇聚，引力波逐渐增强 |
| 长按中断 | 提取失稳 | 进度回退，粒子回落，页面恢复 |
| 提取完成 | 核心脱离作品 | 抽象能量核悬浮，镜头开始拉远 |
| 等待第二次确认 | 核心尚未归属 | 核心停留在私人星系外侧 |
| 提交中 | 等待观测站锁定 | 核心在轨道外等待信号 |
| 投票成功 | 核心入轨 | 核心成为用户行星的卫星 |
| 轨道已满 | 等待交换 | 新核心停留在星系外，旧核心可被选择 |
| 替换提交中 | 核心交接 | 新旧核心在轨道边缘短暂共振 |
| 撤回成功 | 归还核心 | 核心沿航线离开个人星系 |
| 网络超时 | 信号未锁定 | 15 秒后核心主动返航并退出仪式 |

### 4.1 与无垠星空体系的关系

公共星空、个人星图和个人星系使用同一套天文材质、镜头语言和作品光谱，但承担不同的数据职责：

| 空间层 | 作用 | 可见性 |
| --- | --- | --- |
| 公共星空 | 固定作品坐标、自由探索、公共结果 | 所有人共享 |
| 个人星图 | 已解析作品与“我已投票”的私人标记 | 仅当前用户界面 |
| 个人星系 | 用户行星与最多三颗可能性核心 | 仅登录本人 |

个人星系不是公共三维坐标中的隐藏地点。镜头连续表达的是叙事关系，不意味着其他用户可以飞到个人星系。进入仪式前必须保存公共摄像机上下文，退出后依据用户选择回到来源作品详情、来源作品观测壳层或个人星系。

## 5. 新增投票仪式

### 5.1 进入条件

- 未登录用户点击“投出一票”时，先完成姓名、队伍、邮箱和验证码验证。
- 验证完成后回到原作品，并继续进入仪式，不要求再次查找作品。
- 已登录用户直接进入仪式。
- 自投命中、投票未开始、投票已截止或作品不可投时，不启动提取。

### 5.2 仪式前奏

用户点击“投出一票”后，作品详情不立即消失。页面先完成一次短暂的观测锁定：

- 作品行星或作品媒体仍是主视觉。
- 其他非关键界面降低存在感。
- 长按区域、三秒进度和关键说明保持稳定清晰。
- 明确显示作品名称与队伍，避免用户在仪式中忘记自己正在支持谁。

### 5.3 三秒提取

探索者持续按住提取区域三秒。

提取过程中：

- 粒子从行星表面、作品画面边缘和光谱信号中向提取点汇聚。
- 页面背景、行星与边缘界面逐渐受到引力波影响，产生克制的空间扭曲。
- 长按区域、提取进度、作品身份和操作文字不参与扭曲。
- 提取音效逐渐积累，但不能遮盖游戏演示或形成突然高音量。
- 支持的手机设备提供逐渐加深的轻微震动。

中途松开时：

- 提取立即停止。
- 进度平滑回退到零。
- 粒子返回作品行星。
- 引力扭曲快速衰减并完全恢复。
- 不创建选票，也不产生网络写入。

### 5.4 核心脱离与镜头拉远

三秒完成后，核心从作品行星中脱离。脱离瞬间使用一次更明确但短促的声音与触觉反馈。

镜头随后沿当前背景星场逐渐拉远。作品行星不通过硬切消失，而是在同一空间中退到背景，象征视角从公共作品空间过渡到仅本人可见的个人星系层，用户行星与三个等权轨道槽位逐渐显现。

转场开始前保存公共摄像机的位置、朝向、航行模式、观测目标和来源作品。视觉可以连续，公共坐标数据与私人星系数据必须保持隔离。

核心悬浮在用户星系外侧。此时选票尚未生效。

### 5.5 第二次确认与入轨

当仍有空槽时：

- 用户点击悬浮核心完成第二次确认。
- 系统自动选择最近的空轨道槽位，不要求用户进行没有意义的槽位排序。
- 核心在服务端确认前停留在轨道外侧。
- 服务端确认成功后，核心进入轨道，此时选票正式生效。

三个槽位必须保持完全等权。视觉上建议使用同一轨道上的三个相位位置，或三个没有内外等级差异的等价位置。不得通过半径、编号、大小或亮度暗示第一票、第二票和第三票。

### 5.6 轨道已满时的替换

当三颗核心已经占满轨道：

1. 新核心完成提取后悬停在个人星系外。
2. 三颗旧核心进入可选择状态，但仍保持原有身份和轨道关系。
3. 用户亲自选择一颗旧核心。
4. 新旧核心移动到轨道边缘，形成短暂交接与光谱共振。
5. 服务端以一次原子操作完成替换。
6. 成功后旧核心返航，新核心进入空出的相位槽位。
7. 失败时旧核心恢复原轨道，新核心继续等待或最终返航。

替换过程中不能先在视觉上移除旧票，再等待新票提交。交接必须表达“旧票仍在，直到交换被确认”。

### 5.7 成功后的余韵

新核心成功入轨时：

- 核心开始以真实、舒缓的速度绕用户行星公转。
- 触发一次明确但克制的入轨声音与触觉反馈。
- 用户行星变得更明亮、更活跃，并吸收核心的部分光谱特征。
- 页面直接说明选票已生效和当前已使用槽位数量，但不显示公开票数。

镜头在个人星系中继续停留约四秒。核心保持日常公转速度，不为了完成半圈而突然加速。

服务端确认入轨后，来源作品在当前用户的公共星空中获得一个克制的私人轨道标记。它只表达“我已投票”，不改变作品公共亮度、尺寸、探测范围或其他用户看到的状态。

四秒后提供三个去向：

- `继续探索`：返回来源作品的公共观测壳层。
- `返回作品`：回到来源作品详情。
- `留在个人星系`：继续查看或管理当前核心。

## 6. 五秒引导规则

仪式不使用常驻教程覆盖画面。用户在当前阶段五秒以上没有操作时，才显示与状态对应的提示。

| 当前状态 | 五秒后提示方向 |
| --- | --- |
| 尚未开始提取 | 按住提取区域三秒 |
| 提取中断 | 持续按住，松开会回退 |
| 核心悬浮且有空槽 | 点击可能性核心，将它送入你的轨道 |
| 核心悬浮且轨道已满 | 选择一颗旧核心完成交接 |
| 网络等待 | 正在等待观测站锁定选票 |
| 个人星系静止浏览 | 靠近或选择核心查看对应作品 |

提示出现后不抢夺焦点，不替用户操作。用户再次行动时提示自动淡出。

## 7. 个人星系

### 7.1 定位

个人星系是一个可以随时从页面打开的私人空间，不只是投票动画的过场。

它承担三个功能：

1. 观赏自己选择的可能性核心。
2. 查看每颗核心对应的游戏与队伍。
3. 发起归还、返回作品行星或继续探索。

### 7.2 用户行星

用户行星由标准化后的姓名与队伍共同生成稳定种子：

- 同一姓名与队伍在不同设备上生成一致的基础形态。
- 邮箱不参与可见形态和公开标识。
- 行星显示姓名与队伍的身份标识，但这些数据仅当前登录者可见。
- 没有核心时，行星保持克制、安静和较低活跃度。
- 每增加一颗核心，行星获得更多光谱活动、表面生命迹象和轨道响应。

### 7.3 核心与作品身份

核心平时只作为抽象光谱天体公转，不常驻显示作品名称。

只有在用户靠近、悬停、键盘聚焦或点击某颗核心时，才显示：

- 游戏名称。
- 制作队伍。
- 返回作品行星。
- 归还核心。

选择“返回作品行星”后，镜头使用公共星空的导航语言前往该作品固定坐标；不得把私人星系坐标写入公共摄像机状态或分享链接。

### 7.4 私密性

- 个人星系、用户行星形态和当前三颗核心只对本人可见。
- 不提供公开访问、排行榜、他人搜索或个人星系分享链接。
- 公共星图不得根据任何用户的个人选择改变作品亮度。
- 当前登录者可以在公共星空中看见自己的私人投票标记，但该标记不得进入公开接口、分享图或其他用户界面。

### 7.5 截止与结果

投票截止后，个人星系仍可查看，但归还和替换入口被锁定。

最终“宇宙点亮”不会改变个人星系。获奖核心不额外变亮，落选核心不衰减。个人星系只记录“我曾经相信哪些可能性”，不服从最终排名。

## 8. 归还仪式

### 8.1 从个人星系归还

用户点击轨道中的旧核心后，显示一个直接、明确的“归还”确认。归还不需要再次持续三秒。

确认成功后：

- 服务端先完成撤票。
- 核心解除轨道关系并沿原航线返航。
- 用户行星平滑失去由该核心贡献的部分光谱与活跃度。
- 对应槽位恢复为空闲。
- 对应作品的私人投票标记在服务端确认后移除。

### 8.2 从作品详情撤回

作品详情里的“撤回这一票”采用更直接的确认流程，不强制先进入个人星系。成功后仍可用一段简短返航动画表达核心已归还。

### 8.3 撤回失败

如果服务端未确认撤回，核心必须留在原轨道，不能先播放完整返航。页面应说明未能归还，允许用户重试。

## 9. 视觉与材质

### 9.1 可能性核心

核心由三层组成：

1. 内层光谱压缩：来自作品封面与视频的低饱和主色。
2. 中层引力透镜：扭曲经过核心附近的星光与坐标线。
3. 外层粒子包络：粒子密度随提取、提交和入轨状态变化。

核心不能出现切面宝石、金属徽章、火焰魔法球或抽卡稀有度表达。

### 9.2 光谱和谐规则

每颗核心保留来源作品的光谱颜色与纹理，但进入用户行星时必须经过统一调色：

- 限制最大饱和度，避免三颗核心形成廉价彩虹。
- 保持核心之间相近的明度区间，让三票视觉等权。
- 从作品素材提取一组主色与辅助色，只允许少量辅助色进入粒子层。
- 用户行星吸收的是混合后的光谱活动，不直接贴入游戏封面。
- 三颗核心同时存在时，以共享的中性色和赛事强调色统一画面。

### 9.3 引力波

引力波只影响：

- 作品行星或作品媒体。
- 背景星场和坐标网格。
- 非关键边缘界面。

引力波不影响：

- 长按操作区域。
- 三秒提取进度。
- 作品名称与队伍。
- 关键状态和错误信息。

## 10. 声音与触觉

### 10.1 声音

- 长按开始时出现极轻的低频提取底音。
- 随进度增加叠加少量光谱颗粒声。
- 核心脱离和成功入轨各有一次短促确认音。
- 网络等待使用低存在感的持续信号，不使用警报声。
- 声音不能成为唯一反馈，并提供静音控制。

### 10.2 手机触觉

- 提取过程中震动由轻到深，频率或强度随进度上升。
- 核心脱离行星时提供一次明确反馈。
- 核心成功入轨时再提供一次明确反馈。
- 不支持触觉反馈的浏览器自动忽略，不显示错误。
- 触觉不能替代可见进度和文字状态。

## 11. 网络等待与恢复

用户点击悬浮核心后，核心停留在轨道外等待服务端确认。请求在十五秒内使用同一个操作标识进行多次安全重试。

如果十五秒内始终没有确认：

1. 停止重试。
2. 核心主动返回作品行星。
3. 退出仪式并说明本次未能确认。
4. 网络恢复后重新同步服务端选票，避免客户端与真实选票不一致。

网络等待期间不显示通用加载圆环。使用核心呼吸、轨道信号和直接文字表达等待状态。

## 12. 无障碍与性能降级

### 12.1 输入等价

- 鼠标和触控使用持续按住三秒。
- 键盘使用持续按住空格键或 Enter 键三秒。
- 松开对应输入时进度回退。
- 长按区域必须拥有可读名称、当前进度和完整键盘焦点状态。

### 12.2 减少动态效果

开启减少动态效果或设备进入简化二维模式时：

- 保留三秒长按和可见进度。
- 保留数量降低的粒子汇聚。
- 保留少量提取音效和可选触觉反馈。
- 移除强烈引力扭曲和大幅镜头景深变化。
- 使用透明度、缩放和位置变化表达核心提取、入轨与归还。

### 12.3 性能边界

- 粒子数量根据性能等级分级。
- 所有持续运动优先只修改 `transform` 和 `opacity`。
- 仪式关闭后立即停止粒子、轨道帧循环、音频节点和触觉计划。
- 页面切入后台时暂停非必要动画，恢复后根据服务端状态重建画面。

## 13. 关键文案方向

世界观文案和功能文案并存，但操作必须直接可理解。

建议文案：

- 开始：`提取可能性核心`
- 长按提示：`持续按住 3 秒`
- 中断提示：`提取已回退`
- 悬浮提示：`点击核心，将它送入你的轨道`
- 满槽提示：`选择一颗旧核心完成交接`
- 网络等待：`正在等待观测站锁定选票`
- 成功：`可能性核心已入轨，这一票已经生效`
- 归还：`归还可能性核心`
- 超时：`信号未能锁定，核心已经返回`

避免使用：

- “献祭”“誓约”“永久选择”等暗示不可撤回的词。
- “第一选择”“第二选择”“第三选择”等暗示票权排序的词。
- “稀有”“史诗”“传说”等抽卡稀有度词汇。

## 14. 反目标

- 不用普通确认弹窗取代空间仪式。
- 不让长按完成后自动投票。
- 不在网络确认前假装核心已经入轨。
- 不让背景扭曲影响进度、文字和操作区域。
- 不通过核心大小、轨道半径或颜色强度表达票权等级。
- 不公开用户行星、个人星系或选票组合。
- 不让最终获奖结果改写个人星系。
- 不在低性能模式中删除确认步骤或选票管理能力。

## 15. 设计验收问题

1. 用户是否能在不阅读说明书的情况下理解“按住提取，再点击入轨”？
2. 三秒内是否始终知道进度、目标作品和松手后果？
3. 提取完成但未点击核心时，是否明确知道选票尚未生效？
4. 轨道已满时，是否能理解自己正在替换哪一票？
5. 网络等待和网络失败是否不会制造假成功？
6. 三个槽位是否始终视觉等权？
7. 个人星系是否具有私人纪念感，而不是普通选票列表？
8. 不同作品颜色进入同一用户行星后是否和谐、自洽？
9. 减少动态效果模式是否仍保留完整确认意义？
10. 最终结果发布后，个人星系是否仍忠实保存用户原本的选择？
11. 从作品星空进入投票仪式再继续探索时，是否始终感到处于同一宇宙，而不是跳转到另一套界面？
12. 私人投票标记是否严格只对本人可见，且不会破坏公共作品的视觉等权？
