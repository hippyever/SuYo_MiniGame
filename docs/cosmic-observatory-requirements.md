# 溯造 MiniGame 游戏开发大赛：宇宙观测站产品需求

状态：星空体系重构需求已确认
版本：2.0
日期：2026-07-14
关联设计文档：`docs/cosmic-observatory-design.md`

## 1. 产品目标

建立一个公开的 MiniGame 赛事展厅与玩家之声投票系统。用户直接置身于无可见边界的三维共享宇宙，在核心星域中漫游、探测并登陆 10 到 20 款 Demo，再从作品行星提取可能性核心，通过连续空间转场完成最多三张平等票的表达。

产品需要同时完成四件事：

1. 让每款 Demo 被当作一个值得探索的独立世界。
2. 在自由探索和作品完整曝光之间取得平衡。
3. 让投票具有仪式感，但身份验证不重复打断用户。
4. 在结果发布后形成可纪念、可分享的玩家之声星座。
5. 让发现作品、认识作品、投出可能性核心和结果点亮属于同一套空间叙事。

## 2. 非目标

- 不建设游戏平台账号体系或长期社区。
- 不提供游戏在线运行容器，下载和外部试玩仍由作品地址承担。
- 不提供实时票数、点赞、评论或作品热度排行。
- 不同步跨设备探索记录、镜头位置或自动浏览进度。
- 不要求管理员实时推送结果动画到已打开页面。
- 不要求队伍上传专门的行星材质。
- 不建设真实天文模拟器或六自由度飞行游戏；航行服务于作品发现与阅读。
- 不复制有限作品来伪造无限内容，也不允许用户看见星场贴图或移动范围的边界。
- 不公开个人星系、私人投票标记或任何用户的探索进度。

## 3. 用户角色

### 3.1 访客

- 无需登录即可浏览所有作品和完整详情。
- 可以自由探索、自动浏览和搜索作品。
- 尝试投票时需要完成登录验证。

### 3.2 已验证投票者

- 身份由姓名、队伍、邮箱和 6 位邮箱验证码组成。
- 可以投出 1 到 3 张平等票。
- 截止前可以新增、撤回或替换选票。
- 换设备登录后同步当前选票。

### 3.3 管理员

- 管理作品、制作人员、素材和赛事时间。
- 查看、搜索、导出和删除违规投票身份。
- 复核自投和其他异常行为。
- 处理第二个获奖席位的同票裁定。
- 手动发布最终结果。

## 4. 核心规则

| 规则 | 定义 |
| --- | --- |
| 作品数量 | 典型 10 到 20 款 |
| 投票资格 | 任何完成邮箱验证的访问者 |
| 每人票数 | 1 到 3 票 |
| 票权 | 每票平等，不排序、不加权 |
| 修改 | 截止前可随时新增、撤回或替换 |
| 自投 | 登录姓名匹配作品制作人员姓名时禁止 |
| 公开热度 | 投票期内禁止显示 |
| 获奖数量 | 两项同等级玩家之声奖 |
| 星座范围 | 前五名，边界同票时扩展 |
| 获奖同票 | 第二个席位同票时管理员裁定一款 |
| 结果发布 | 截止后锁定，管理员复核后手动发布 |
| 空间坐标 | 所有用户共享稳定三维作品坐标 |
| 空间语义 | 作品位置自由散布，不表达类型、团队、作者或时间关系 |
| 探测持久性 | 已解析颜色与身份仅在当前设备保存 |

## 5. 产品状态机

```text
UPCOMING 投票未开始
  -> OPEN 投票进行中
  -> LOCKED 投票已截止，结果隐藏
  -> REVIEW 违规票与同票复核中
  -> PUBLISHED 结果已发布
```

状态要求：

| 状态 | 浏览作品 | 新增或修改投票 | 显示票数 | 管理员删票 | 发布结果 |
| --- | --- | --- | --- | --- | --- |
| UPCOMING | 是 | 否 | 否 | 否 | 否 |
| OPEN | 是 | 是 | 否 | 是 | 否 |
| LOCKED | 是 | 否 | 否 | 是 | 否 |
| REVIEW | 是 | 否 | 否 | 是 | 是 |
| PUBLISHED | 是 | 否 | 是 | 默认禁止 | 已完成 |

## 6. 功能需求

### 6.1 动态海报

#### FR-POSTER-01 每次进入显示

每次页面打开或刷新都显示完整动态赛事海报，不使用“仅首次访问”逻辑。

验收：

- 首帧可读取赛事口号、名称、主题和工作室标识。
- “开始观测”从首帧即可点击。
- 点击后进入星图，不存在无法跳过的等待阶段。

#### FR-POSTER-02 正式内容

必须显示：

- 口号：溯求本源，造物不止
- 赛事名称：溯造 MiniGame 游戏开发大赛
- 主题：宇宙
- SuYo Studio 溯游工作室标识
- 操作：开始观测

#### FR-POSTER-03 深链接

通过作品分享链接进入时仍显示海报。点击“开始观测”后自动导航到目标行星。

### 6.2 共享宇宙与星图

#### FR-MAP-01 固定公共三维坐标

所有用户必须看到相同的作品三维坐标。坐标不能被访问量、票数、当前用户、设备随机数或探索状态改变。

建议生成规则：

```text
coordinate3D = layout3D(eventSeed, gameId, allGameIds)
planetSeed = hash(eventSeed + gameId)
```

管理员发布作品集合后生成并保存坐标。作品增删时必须避免静默重排已有坐标。

#### FR-MAP-02 三维自由散布

- 作品必须分布在具有前后、上下和左右关系的空间体积内，不得共享同一个或近似相同的深度平面。
- 作品位置不表达类型、主题、团队、作者、时间或热度关系。
- 布局必须保证在全景观测状态下全部作品异常信号可见且可区分。
- 任意两个作品在其观测壳层附近都不得永久互相遮挡到无法选择。

#### FR-MAP-03 初始置身核心星域

点击“开始观测”后，摄像机默认位于核心星域内部。初始视野之外的前后左右仍必须存在作品异常信号，初始画面不得突出任何指定作品。

#### FR-MAP-04 航行控制

桌面必须支持：

- 鼠标滚轮沿当前观察方向前进或后退。
- 按住鼠标中键拖动进行上下、左右侧移。
- 按住鼠标左键拖动旋转摄像机。
- 点击近距离已解析作品进入详情。

移动端必须支持：

- 单指拖动旋转摄像机。
- 双指捏合沿观察方向前进或后退。
- 双指同向拖动进行侧移。

键盘必须提供前进、后退、旋转、侧移、进入、返回和全景复位的完整等价操作。前进与后退按键以及界面 `+ / −` 控件必须支持长按连续航行，并在松开、页面失焦或切入后台时立即停止。页面必须阻止浏览器滚动、上下文菜单或默认手势在星空激活时错误抢占控制。

#### FR-MAP-05 全景观测状态

- 用户持续后退到最小缩放附近时，系统必须连续进入全景观测状态。
- 系统在过渡中逐渐校正摄像机位置和朝向，把全部作品异常信号框入视野，不得硬切或瞬移。
- 未解析作品在全景状态下只显示异常亮点，不显示名称、团队、作者、封面或远程进入入口。
- 已解析作品保留颜色与身份标记，但标签必须执行碰撞避让和密度降噪。
- 用户再次前进、旋转或侧移时退出全景状态并恢复自由航行。

#### FR-MAP-06 无限背景连续性

- 在最小缩放、最大允许推进距离、任意横向方向和任意屏幕比例下，用户都不得看见星点矩形边缘、纹理接缝、空白区域或地图边界。
- 背景星场必须随摄像机持续生成，并提供至少三个具有不同视差响应的深度层级。
- 摄像机远离数值原点时必须通过浮动原点或等价方案保持渲染稳定；重定位不得改变作品公共坐标语义或产生可见跳变。
- 不得复制作品天体、改变作品固定坐标或通过可感知的瞬间传送制造无限效果。

#### FR-MAP-07 无感返航引导

当用户持续远离全部作品且没有搜索或自动巡航目标时，系统必须逐渐把航向引回核心星域：

- 不显示边界警告、回弹动画或“已到地图尽头”。
- 修正连续、可逆，并低于正常用户输入的控制权重。
- 用户做出反向输入时立即服从用户操作。
- 搜索、自动巡航和深链接导航期间不得被返航引导改写目标。

#### FR-MAP-08 视觉等权

投票结果发布前：

- 所有作品天体的基础尺寸和异常信号亮度处于同一规则范围。
- 运动幅度和特效复杂度不能由热度决定。
- 初始视野、布局和探测半径不能为某款作品提供主推位置。
- 不显示访问量、票数、排名或“热门”标签。

#### FR-MAP-09 距离揭示顺序

系统必须根据摄像机与作品的空间距离连续输出以下阶段：

1. 深空异常：只显示耀眼光点，不显示身份，不可点击。
2. 光谱初判：显示作品颜色与类型，不可点击。
3. 玩法解析：显示封面与静音玩法片段，不可进入详情。
4. 身份解析：显示作品名称，可聚焦。
5. 来源解析：显示团队与作者，可点击查看游戏。
6. 登陆：显示完整作品内容、成员、下载与投票入口。

阶段切换必须以连续插值和稳定阈值完成，不得因悬停而提前泄露信息，也不得在阈值边缘来回闪烁。

#### FR-MAP-10 被动多目标探测

- 探测器必须自动解析范围内的全部作品，不要求使用中心准星逐个锁定。
- 多目标标签必须锚定各自天体，并具备碰撞避让、引线和优先级。
- 标签优先级依次为：当前观测壳层目标、当前搜索或巡航目标、距离更近目标、其他目标。
- 标签避让不得改变天体真实屏幕位置或使用户误判方向。

#### FR-MAP-11 已解析状态持久化

- 作品完成身份解析或进入详情时，当前设备必须保存其已解析状态。
- 已解析作品退远后仍保留作品颜色与身份标记。
- 刷新和重新打开页面后恢复已解析状态；换设备登录后不得同步。
- 清除本地探索记录时一并清除已解析状态，但不得改变选票。

#### FR-MAP-12 离屏异常提示

- 屏幕边缘必须用克制的方向刻度、微弱光谱脉冲或数量提示表达视野外作品。
- 未解析作品提示只能包含方向、距离层级和聚合数量，不得暴露身份。
- 同一方向上的多个目标必须聚合，转入视野时提示自然收束到真实天体。
- 提示不得持续高频闪烁，不得遮挡全局功能和作品标签。
- 辅助技术必须能获得等价文本，例如“右侧有 3 个未知信号”。

#### FR-MAP-13 恒星到行星的连续转化

远距离作品恒星与近距离作品行星必须是同一作品对象的连续表现。光谱、外观和媒体由赛事种子、作品 ID、封面与视频稳定生成，不得在接近阈值处突然替换为无关模型。

#### FR-MAP-14 观测壳层与接近节奏

- 从用户对准一个典型远方异常信号并持续前进，到进入可阅读、可点击的来源解析阶段，目标时长为 3 到 5 秒。
- 当航向进入作品捕获范围时，前进速度必须连续降低，并稳定在观测壳层，避免滚轮或捏合过冲。
- 自动减速不得锁死摄像机；旋转、侧移和反向前进输入必须继续有效。
- 多个捕获范围重叠时，以用户航向最接近且距离更近的目标作为减速参考，不得在目标间抖动。

#### FR-MAP-15 行星自动生成

行星由赛事种子、作品 ID、封面色彩和纹理、可用时的视频画面稳定生成。同一作品多次访问必须生成相同的远方光谱、近处行星与可能性核心色谱。系统不能要求队伍额外上传行星材质。

#### FR-MAP-16 性能模式

系统必须支持完整、标准和简化三种表现等级。标准模式保留真实三维坐标；简化模式可使用二维投影模拟纵深，但必须保留旋转、推进、全景、探测、登陆和投票功能。性能降级不能改变公共坐标、内容、选票或结果。

#### FR-MAP-17 控制权仲裁

多个摄像机行为同时存在时必须按以下优先级仲裁：

1. 用户主动旋转、推进、后退或侧移。
2. 用户明确触发的返回、退出或取消。
3. 搜索、深链接或自动巡航导航。
4. 观测壳层减速。
5. 无感返航引导。

低优先级行为不得与高优先级行为叠加造成抖动、目标跳换或控制迟滞。

### 6.3 视频预览

#### FR-PREVIEW-01 单目标播放

探测器可以同时显示多个作品的封面信号，但同一时间最多允许一个行星解码并播放静音动态预览。动态目标优先选择距离最近且屏幕可见面积最大的作品，切换前必须停止旧目标。

#### FR-PREVIEW-02 资源释放

离开玩法解析范围后必须暂停视频、取消后续下载并释放解码资源。

#### FR-PREVIEW-03 降级

以下情况使用封面动态替代：

- 用户启用节能观测模式。
- 系统判断设备性能不足。
- 网络数据节省模式开启。
- 作品未上传视频。
- 浏览器拒绝自动播放。

### 6.4 作品登陆视图

#### FR-DETAIL-01 连续空间转场

从来源解析状态进入详情时使用镜头推进和表面穿越，不打开独立弹窗。退出后恢复进入前的摄像机位置、朝向、航行模式和观测目标。

减少动态模式使用即时详情切换，不执行快速穿越。

#### FR-DETAIL-02 内容顺序

详情按以下顺序呈现：

1. 玩法画面或封面。
2. 游戏名称。
3. 游戏简介。
4. 创作手记。
5. 队伍与制作人员。
6. 完整视频、下载和投票操作。

#### FR-DETAIL-03 制作人员卫星

每位制作人员支持：

- 姓名，必填。
- 头像，可选但推荐。
- 职责，可选。
- 展示顺序，必填。

详情视图将制作人员表现为围绕作品的卫星。选择卫星后显示姓名和职责。

#### FR-DETAIL-04 创作手记

每款作品支持创作手记，建议 40 到 80 字。游戏简介与创作手记为两个独立字段。

### 6.5 搜索

#### FR-SEARCH-01 检索字段

支持按以下内容模糊检索：

- 游戏名称。
- 队伍名称。
- 制作人员姓名。

#### FR-SEARCH-02 结果内容

搜索结果直接显示游戏名称、封面缩略图、队伍和坐标。

#### FR-SEARCH-03 导航

选择搜索结果后，以明确导航信标和连续三维自动航行把镜头移动到目标行星的观测壳层，不得硬切。搜索命中本身不写入已解析或已探索状态；到达对应距离阶段后按正常探测规则记录。

### 6.6 自动浏览全部作品

#### FR-ROUTE-01 本地随机顺序

每台设备生成一个覆盖全部作品的随机不重复顺序。顺序与进度保存在本地。

#### FR-ROUTE-02 稳定恢复

刷新或关闭后再次进入时继续原顺序，不重新洗牌。作品集合变化时保留仍存在作品的相对顺序，并将新作品随机插入未完成部分。

#### FR-ROUTE-03 巡航控制

自动巡航沿三维航线到达下一作品的观测壳层后暂停并交还控制。支持开始、暂停、继续、跳过当前作品和退出自动浏览。用户主动旋转、侧移或反向推进时，巡航必须暂停而不是与输入争夺控制。

#### FR-ROUTE-04 非强制完成

用户不需要完成全部作品才能投票。我的投票中显示未探索数量，但只提供提醒，不阻止提交。

### 6.7 本地探索数据

#### FR-LOCAL-01 保存范围

仅在当前设备保存：

- 已探索作品 ID。
- 已解析作品 ID。
- 自动浏览顺序与进度。
- 最近公共摄像机位置、朝向、航行模式和观测目标。
- 性能模式偏好。

#### FR-LOCAL-02 不跨设备同步

同一投票者换设备登录后不得同步上述探索数据，只同步当前选票。

#### FR-LOCAL-03 清除

提供清除本地探索和已解析记录的能力，不影响服务端选票、个人星系或当前私人投票标记。清除后仍可从服务端选票重建“已投票”状态。

### 6.8 身份验证

#### FR-AUTH-01 延迟登录

未登录用户可浏览所有作品。第一次执行投票或撤票行为时才要求登录。

#### FR-AUTH-02 身份字段

- 姓名，必填。
- 队伍，必填。
- 邮箱，必填。
- 6 位数字验证码，必填。

#### FR-AUTH-03 会话

验证成功后创建当前设备登录会话。后续投票操作不再次要求验证码，除非会话失效或用户退出。

推荐默认会话有效期为 30 天，但不能超过赛事结果发布后的管理策略期限。

#### FR-AUTH-04 操作恢复

用户从某颗行星点击投票后被要求登录，验证成功必须返回原作品并继续投票，不能丢失目标。

#### FR-AUTH-05 开放资格

任何完成邮箱验证的人可以投票，不校验预导入名单或邀请码。

#### FR-AUTH-06 验证码安全

- 验证码 10 分钟有效。
- 连续发送至少间隔 60 秒。
- 按邮箱和来源地址限制小时频率。
- 验证码只保存不可逆摘要。
- 正式环境不得把验证码返回前端或写入公开日志。

### 6.9 选票

可能性核心投票仪式、个人星系、网络恢复与无障碍要求见 `docs/possibility-core-voting-ritual-requirements.md`。本节保留业务规则总览，发生表述差异时以专项需求中的交互状态定义为准，票数和票权规则仍以本文件为准。

#### FR-VOTE-01 平等票

每位投票者可选择 1 到 3 款不同作品。所有选择各计一票，不提供排序。

#### FR-VOTE-02 可修改

OPEN 状态下可以：

- 新增选票。
- 撤回选票。
- 用另一款作品替换选票。

服务端保存当前有效选票集合，并记录每次变更的审计事件。

#### FR-VOTE-03 跨设备同步

同一身份在另一台设备完成登录后，显示服务端当前选票。跨设备同时修改时，以服务端最后成功版本为准，并使用版本号避免静默覆盖。

推荐请求携带 `ballotVersion`，版本冲突返回最新选票并要求用户确认后重试。

#### FR-VOTE-04 自投拦截

系统对登录姓名和全部制作人员姓名进行标准化匹配。标准化至少包括：

- 去除首尾空格。
- 合并连续空格。
- 英文字母不区分大小写。
- 统一常见全角与半角字符。

命中后禁止把该作品加入选票。队伍名称不作为自动拦截条件，只用于后台复核。

#### FR-VOTE-05 未探索提醒

在我的投票或确认变更时显示尚未探索数量，并提供“自动浏览全部作品”。提醒不得阻止操作。

#### FR-VOTE-06 原子更新

一次选票变更必须原子校验并提交：

- 投票状态仍为 OPEN。
- 作品存在且可投。
- 数量为 1 到 3。
- 无重复作品。
- 无自投命中。
- 会话有效。

任何校验失败时不得部分更新。

#### FR-VOTE-07 空间连续入口

- 投票仪式只能从当前作品详情或等价的已确认作品上下文启动。
- 三秒提取必须从当前作品行星或媒体光谱中产生可能性核心，不能显示与来源作品无关的通用投票动画。
- 提取完成后，镜头必须沿当前背景星场连续拉远并显露个人星系，不得切换为普通确认弹窗。
- 进入个人星系前必须保存公共摄像机位置、朝向、航行模式、观测目标和来源作品。
- 减少动态效果模式可以缩短转场，但必须保留“作品空间 → 核心提取 → 个人星系”的因果关系。

#### FR-VOTE-08 个人星系返回航迹

- 投票成功后提供“继续探索”“返回作品”和“留在个人星系”三个明确去向。
- “继续探索”返回来源作品的公共观测壳层并恢复合理朝向。
- “返回作品”回到来源作品详情。
- 从个人星系的旧核心选择“返回作品行星”时，导航到该作品固定公共坐标。
- 返回过程不得把私人星系坐标写入公共摄像机状态或分享链接。

#### FR-VOTE-09 私人投票标记

- 服务端确认选票生效后，当前用户的公共星空可以在对应作品旁显示克制的私人轨道标记。
- 标记只对当前登录用户可见，不得进入公共作品接口、分享图或其他用户界面。
- 标记不得改变作品基础尺寸、亮度、探测半径、布局或自动巡航顺序。
- 撤回或替换只有在服务端确认后才移除旧标记、增加新标记；失败时维持原状态。

#### FR-VOTE-10 三层数据边界

- 公共宇宙只包含固定作品坐标、公共赛事状态和已发布结果。
- 个人星图包含当前设备已解析状态、探索进度和当前登录者的私人投票标记。
- 个人星系由服务端当前选票重建，只允许登录本人访问。
- 结果发布只改变公共宇宙，不得改变个人星系中核心的亮度、光谱、大小或轨道。

### 6.10 作品分享

#### FR-SHARE-01 稳定链接

每款作品拥有稳定的可分享路径或查询参数，不依赖当前星图镜头位置。

#### FR-SHARE-02 分享卡片

Open Graph 分享信息直接展示：

- 游戏名称。
- 游戏封面。
- 队伍名称。
- 赛事名称。

#### FR-SHARE-03 进入行为

分享接收者打开链接后先看动态海报，点击“开始观测”后自动导航到对应行星。

### 6.11 投票锁定与结果发布

#### FR-RESULT-01 自动锁定

到达投票截止时间后自动进入 LOCKED 状态，禁止全部新增、撤回和替换操作。

#### FR-RESULT-02 结果隐藏

LOCKED 和 REVIEW 状态不向公开接口返回票数、排名、候选关系或任何可推导结果的数据。

#### FR-RESULT-03 两项同等级奖

发布结果时明确两款同等级玩家之声获奖作品。前台不得使用冠军、亚军或第一、第二的等级表达。

#### FR-RESULT-04 可扩展星座

按有效票数取前五名。若第五名边界有同票，全部同票作品进入星座。

示例：

```text
票数：12, 10, 8, 6, 5, 5, 3
星座节点：前 6 款
```

#### FR-RESULT-05 获奖同票裁定

当两个获奖席位的边界存在同票候选时：

- 后台显示全部候选及有效票数。
- 票数严格高于边界票数的作品自动占据席位。
- 管理员从边界同票候选中选出剩余所需数量，使获奖作品总数严格为两款。
- 系统记录裁定人、时间、候选列表、最终选择和备注。
- 未裁定前禁止发布结果。

如果三款及以上作品并列最高票，管理员从这些作品中选择两款。最终两款作品获得同等级玩家之声奖。

#### FR-RESULT-06 手动发布

管理员完成复核后手动发布。发布前显示最终预览和二次确认。

已打开页面不要求实时切换。刷新、重新打开或下一次公开数据请求时进入 PUBLISHED 状态。

#### FR-RESULT-07 发布后锁定

结果发布后默认禁止管理员删除选票或修改裁定。确需更正时必须进入明确的“撤回结果”流程并保留审计记录，不能静默改变结果。

### 6.12 管理后台

#### FR-ADMIN-01 游戏管理

支持创建、编辑、排序、发布、下架和删除游戏。

字段包括：

- 名称。
- 队伍。
- 封面。
- 游戏简介。
- 创作手记。
- 标签。
- 视频文件或链接。
- 下载地址。
- 制作人员数组。
- 行星种子与固定坐标，只读展示，管理员可显式重新生成。

#### FR-ADMIN-02 制作人员管理

支持逐人添加姓名、头像、职责和顺序。姓名用于自投匹配，删除或修改时需提示可能影响复核结果。

#### FR-ADMIN-03 投票者管理

支持按姓名、队伍和邮箱搜索。展示当前选票、修改时间、自投疑似关联和操作审计。

#### FR-ADMIN-04 删除违规投票者

删除对象是投票身份及其全部有效票，不是单独某一票。操作需要：

- 二次确认。
- 必填删除原因。
- 记录管理员、时间和删除前选票。
- 立即重新计算未发布结果。

#### FR-ADMIN-05 导出

支持导出有效选票和审计记录。公开前台不得访问导出接口。

#### FR-ADMIN-06 结果复核

后台提供：

- 有效投票者数。
- 总有效票数。
- 作品票数。
- 星座边界同票情况。
- 获奖席位同票情况。
- 管理员裁定入口。
- 发布前结果预览。

## 8. 接口方向

- `GET /api/site`：赛事设置、公开作品、当前状态、已发布结果。

- `GET /api/ballot/me`：读取当前选票。
- `PUT /api/ballot/me`：原子替换当前选票集合。
- `POST /api/auth/logout`：退出当前设备会话。

- `POST /api/admin/games`

## 9. 非功能需求

### 9.1 性能

- 首屏海报 LCP 小于 2.5 秒。
- 常见 4G 下星图 4 秒内可操作。
- 完整模式目标 60 FPS，降级模式稳定不低于 30 FPS。
- 同时只解码一个预览视频。
- 背景星场分块生成、回收和浮动原点重定位不得产生可见停顿、接缝或单帧跳变。
- 多目标标签的布局计算不得触发逐目标 DOM 高频重排；标签数量超过可读密度时必须聚合或降噪。
- 大型渲染库与详情媒体延迟加载。
- 低端设备自动降级，不能崩溃或显示空白 Canvas。

### 9.2 可用性

- 移动端与桌面端均可完成所有核心任务。
- 所有投票变更有成功、失败和冲突反馈。
- 验证登录后恢复用户原操作。
- 返回星图后恢复原位置。
- 自动浏览可随时暂停和退出。
- 用户在任意航向主动输入时拥有高于返航引导、观测减速和自动巡航的控制优先级。
- 公共星空、作品详情、投票仪式和个人星系之间的返回目标必须确定，不得丢失来源作品。

### 9.3 无障碍

- 键盘可访问星图索引、作品详情、登录和投票。
- 键盘可完成三维旋转、前后推进、侧移、全景观测和目标进入。
- 辅助技术可以读取离屏异常信号的方向、距离层级与聚合数量。
- 触摸目标至少 44 x 44 像素。
- 正文与表单达到 WCAG AA。
- 动画遵循 `prefers-reduced-motion`。
- 不使用颜色作为唯一状态提示。
- 简化二维模式同时作为渲染失败时的可访问后备。

### 9.4 安全与隐私

- 公开接口不返回邮箱、验证码、来源地址或后台审计数据。
- 管理员密码必须在生产环境使用高强度随机值。
- 管理接口必须限速并使用安全会话。
- 上传文件校验类型、大小与文件名，禁止执行上传内容。
- 验证码、会话和管理操作具有服务端过期与撤销能力。
- 删除违规票、裁定和发布结果全部写入不可静默覆盖的审计日志。
- 邮箱只用于本次投票验证，不用于营销。

建议默认在结果发布 30 天后匿名化邮箱和来源地址。正式保留期限由主办方在上线前确认，并写入隐私说明。

### 9.5 兼容性

- 支持当前主流 Chrome、Edge、Safari 和移动浏览器。
- WebGL 不可用时自动进入简化二维模式。
- 浏览器拒绝视频自动播放时使用封面替代。
- 页面在 320px 宽度下仍可完成登录和投票。

## 10. 后台复核规则

### 10.1 违规投票删除

管理员选择投票者，填写原因，二次确认后删除其全部有效票。系统保留匿名化审计事件和删除前选票快照，仅对管理员可见。

### 10.2 自投人工复核

自动规则只按制作人员姓名匹配。队伍名称、联合制作、别名和疑似关联由管理员人工复核。

### 10.3 获奖席位同票

示例：

```text
A 作品：12 票
B 作品：10 票
C 作品：10 票
```

A 自动成为获奖候选。B 与 C 进入第二席位裁定。管理员选择 B 或 C，最终 A 和被选作品获得同等级玩家之声奖。如果 A、B、C 三款均为 12 票，则管理员从三款中选择两款。

### 10.4 星座边界同票

示例：

```text
A 12, B 10, C 8, D 7, E 6, F 6, G 4
```

A 到 F 全部进入星座，共六个节点，不需要管理员裁定。

## 11. 端到端验收场景

### AC-01 未登录探索

用户打开网站，看到动态海报，点击“开始观测”后置身核心星域内部，旋转镜头、前后推进和侧移，接近并进入任意作品详情，全程不要求登录。

### AC-02 登录后继续投票

未登录用户在作品详情点击“投票”，完成姓名、队伍、邮箱和验证码验证，验证成功后返回原作品并完成投票，没有再次查找作品。

### AC-03 最多三票且等权

用户可选择 1 到 3 款不同作品。第四款无法加入。界面和结果计算都不提供排序或权重。

### AC-04 截止前修改

用户投出三票后刷新、重新登录或换设备，仍能读取当前选票，并在截止前撤回其中一票或替换为另一款。

### AC-05 探索记录不跨设备

用户在手机探索五款作品并登录投票，随后在电脑登录。电脑同步选票，但已探索数量、镜头和自动浏览进度为空。

### AC-06 自投拦截

登录姓名与某作品制作人员姓名标准化后相同。该作品的投票操作被禁用，其他作品不受影响。

### AC-07 自动浏览公平

两台新设备获得不同的作品顺序，各自顺序无重复并覆盖全部作品。刷新后每台设备保持自己的原顺序。

### AC-08 投票期间无热度暗示

无论作品票数多少，公开星图都不改变行星尺寸、亮度和标签。公开接口不返回票数。

### AC-09 截止锁定

到达截止时间后，已登录用户无法新增、撤回或替换选票。页面显示结果复核中，不显示排名。

### AC-10 管理员删票

管理员删除违规投票者后，其全部票数被撤销，未发布结果重新计算，操作原因和管理员身份进入审计记录。

### AC-11 同票裁定

第二个获奖席位存在同票时，后台阻止发布。管理员完成裁定并填写备注后才允许发布。

### AC-12 星座扩展

第五名边界存在同票时，所有边界同票作品进入结果星座，节点数量可以超过五个。

### AC-13 手动发布结果

管理员发布前，公开页面始终隐藏结果。发布后，用户刷新或重新进入，经过动态海报后看到两颗同等级获奖恒星和最终星座。

### AC-14 性能降级

低性能或不支持 WebGL 的设备自动进入简化模式，仍能以二维投影视觉完成旋转、推进、全景观测、搜索、自动浏览、作品详情、登录、投票和结果查看。

### AC-15 无可见边界

用户后退到最小缩放、持续向同一方向推进并进行大幅侧移。背景繁星持续生成，没有矩形边缘、纹理接缝、空白、硬回弹、边界文案或可见瞬移。

### AC-16 三维纵深

用户从核心星域内部旋转观察时，可以在前后左右发现不同作品信号。向一颗作品推进后，其他作品因视锥、距离和遮蔽逐渐退出视野，而不是始终停留在同一缩放平面。

### AC-17 全景观测

用户从任意自由航行位置持续后退。镜头连续校正并进入全景观测状态，全部作品异常信号被框入视野；未知作品不可隔空识别或进入，已解析作品保留颜色与身份。

### AC-18 多目标自动探测

三颗作品同时进入探测范围时，系统自动解析三者并将信息锚定在对应天体附近。标签不重叠到无法阅读，也不要求用户用准星逐颗锁定。

### AC-19 信息揭示公平

用户接近未解析作品时，依次看见颜色与类型、封面与静音玩法、名称、团队与作者，最后才可进入详情。刷新后该作品退远仍保留颜色和身份；另一台设备不自动获得此状态。

### AC-20 接近减速

用户持续推进一颗作品，约 3 到 5 秒到达观测壳层。系统连续减速并避免过冲；用户反向滚轮、反向捏合、旋转或侧移仍能立即脱离。

### AC-21 离屏引导

初始视野外存在作品时，屏幕边缘显示克制的方向和聚合数量提示，不暴露未知作品身份。用户转向后，提示收束到实际天体；辅助技术读出等价方向说明。

### AC-22 无感返航

用户持续远离全部作品且没有导航目标时，系统逐渐把航向引回核心星域，没有硬边界反馈。用户主动反向操作后立即获得控制，搜索和自动巡航目标不被改写。

### AC-23 星空投票连续性

用户从已登陆作品启动投票，可能性核心从当前作品光谱中被提取，镜头沿同一背景星场拉远进入私人星系。服务端确认入轨后，选择“继续探索”返回来源作品观测壳层，公共摄像机上下文没有丢失。

### AC-24 私人标记与公共公平

投票成功后只有当前用户看见对应作品的私人轨道标记，其他用户和公开接口均不出现。标记不改变作品亮度、尺寸、探测范围或巡航顺序。撤回失败时标记与个人核心都保持原状。

### AC-25 结果与个人星系分离

发布玩家之声结果后，公共宇宙点亮获奖恒星和结果星座；当前用户的个人星系及三颗可能性核心保持发布前的光谱、亮度和轨道。

## 12. 实施优先级

### P0 业务正确性

- 登录会话。
- 选票读取、修改、撤回和并发版本控制。
- 自投拦截。
- 管理员删票与审计。
- LOCKED、REVIEW、PUBLISHED 状态。
- 获奖同票裁定与结果发布。
- 新游戏和制作人员数据模型。

### P1 核心体验

- 动态赛事海报。
- 固定三维公共坐标。
- 摄像机旋转、推进、侧移与全景观测状态机。
- 无限背景星场与浮动原点。
- 无感返航、观测壳层和离屏异常提示。
- 搜索和自动浏览。
- 连续作品登陆视图。
- 星空到可能性核心投票的连续转场。
- 我的投票与个人星系返回航迹。

### P2 艺术表现

- 自动行星生成。
- 恒星到行星转化与分级信息揭示。
- 静音视频预览。
- 制作人员卫星。
- 投票核心轨道动画。
- 已解析状态与私人投票标记。
- 最终星座与获奖恒星。

### P3 优化与传播

- 完整 WebGL 表现。
- 自动性能分级。
- 分享卡片和结果分享图。
- Lighthouse、移动设备矩阵和视觉回归测试。

## 13. 上线前待配置项

以下内容不影响设计方向，但上线前必须由主办方填写：

- 投票开始和截止时间。
- 赛事唯一种子。
- 正式管理后台密码。
- SMTP 发信配置。
- 允许的封面、视频和头像大小上限。
- 登录会话有效期。
- 个人数据保留与匿名化期限。
- 管理员名单和结果裁定责任人。
